## Feature: expression-based indexes, partial indexes, and collation support

### Expression-based indexes

`ixLower` and `ixUnaccent` wrap entity properties directly inside `hasIndex` selectors and compose naturally:

```typescript
// Plain columns (unchanged)
entity.hasIndex('ix_posts_query', e => [e.userId, e.publishedAt]);

// Expression columns
entity.hasIndex('ix_name_lower', e => [ixLower(ixUnaccent(e.name))]);
// → CREATE INDEX ix_name_lower ON ... (lower(unaccent("name")))

// Mixed plain + expression columns
entity.hasIndex('ix_mixed', e => [e.id, ixLower(e.name)]);
```

### Partial indexes

`.where()` on `IndexBuilder` adds a `WHERE` clause:

```typescript
entity.hasIndex('ix_active_email', e => [e.email])
  .isUnique()
  .where('active = true');
// → CREATE UNIQUE INDEX ix_active_email ON ... ("email") WHERE active = true
```

### Raw expression escape hatch

`.withExpression()` for complex SQL that can't be expressed with helpers:

```typescript
entity.hasIndex('ix_jsonb')
  .withExpression('(data->>\'key\')::int')
  .where('deleted_at IS NULL');
```

### Collation support

Define custom PostgreSQL collations with `pgCollation()` and attach them to columns with `.hasCollation()`:

```typescript
const ndCiAi = pgCollation({
  name: 'nd_ci_ai',
  provider: 'icu',
  locale: 'und-u-ks-level1',
  deterministic: false,
});

entity.property(e => e.name).hasType(varchar('name', 200)).hasCollation(ndCiAi);
```

Produces:
```sql
CREATE COLLATION "nd_ci_ai" (provider = 'icu', locale = 'und-u-ks-level1', deterministic = false);
-- column definition:
"name" varchar(200) COLLATE "nd_ci_ai" NOT NULL
```

Collation lifecycle:
- **`ensureCreated()`** — creates collations before tables
- **`analyze()`** — detects missing collations and collation mismatches on existing columns
- **`migrate()`** — applies `CREATE COLLATION` and `ALTER COLUMN ... COLLATE` automatically
- **`MigrationScaffold`** — generates `CREATE COLLATION IF NOT EXISTS` / `DROP COLLATION IF EXISTS` in scaffolded files
