## Feature: expression-based indexes and partial indexes

### Overview

Added support for expression-based index columns (e.g., `lower(unaccent(name))`) and partial indexes (`WHERE` clause) in the entity index builder API.

### New API

**Expression helpers in selectors** — `ixLower` and `ixUnaccent` wrap entity properties directly inside `hasIndex` selectors and compose naturally:

```typescript
// Plain columns (unchanged)
entity.hasIndex('ix_posts_query', e => [e.userId, e.publishedAt]);

// Expression columns
entity.hasIndex('ix_name_lower', e => [ixLower(ixUnaccent(e.name))]);
// → CREATE INDEX ix_name_lower ON ... (lower(unaccent("name")))

// Mixed plain + expression columns
entity.hasIndex('ix_mixed', e => [e.id, ixLower(e.name)]);
```

**Partial indexes** — `.where()` on `IndexBuilder` adds a `WHERE` clause:

```typescript
entity.hasIndex('ix_active_email', e => [e.email])
  .isUnique()
  .where('active = true');
// → CREATE UNIQUE INDEX ix_active_email ON ... ("email") WHERE active = true
```

**Raw expression escape hatch** — `.withExpression()` for complex SQL that can't be expressed with helpers:

```typescript
entity.hasIndex('ix_jsonb')
  .withExpression('(data->>\'key\')::int')
  .where('deleted_at IS NULL');
```

### Files Changed

- `src/entity/entity-base.ts` — added `ixLower`, `ixUnaccent` helpers and `IndexColumnRef` type; added `expressions` and `where` fields to `IndexMetadata`
- `src/entity/entity-builder.ts` — refactored `hasIndex` to process returned `IndexColumnRef` array; added `withExpression()` and `where()` to `IndexBuilder`
- `src/schema/table-builder.ts` — added `expressions` and `where` fields to `IndexDefinition`
- `src/migration/db-schema-manager.ts` — expression and WHERE support in `executeCreateIndex`, operation creation, and `describeOperation`
- `src/migration/migration-scaffold.ts` — expression and WHERE support in `buildCreateIndexSql`
- `src/index.ts` — exported `ixLower`, `ixUnaccent`
- `tests/schema/expression-index.test.ts` — new test suite (9 tests)
