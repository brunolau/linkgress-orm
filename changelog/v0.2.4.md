## New Features

### `exists()` Method
Added `exists()` method to query builders for efficiently checking if any rows match a query. More performant than `count() > 0` as it can short-circuit after finding the first row.

```typescript
const hasOrders = await db.orders
  .where(o => eq(o.user!.username, 'alice'))
  .exists();  // Returns boolean
```

- Commit: [7cc76f9](https://github.com/brunolau/linkgress-orm/commit/7cc76f9)

## Bug Fixes

### `eq()` and `ne()` with `null` Values
Fixed `eq(field, null)` generating invalid SQL (`"field" =` with nothing after). Now correctly converts to `IS NULL` / `IS NOT NULL`:

```typescript
// Before: Generated "age =" causing syntax error
// After: Generates "age IS NULL"
db.users.where(u => eq(u.age, null))

// ne() also fixed - generates "age IS NOT NULL"
db.users.where(u => ne(u.age, null))
```

- Commit: [c340f71](https://github.com/brunolau/linkgress-orm/commit/c340f71)

### `count()` with Navigation Properties in WHERE
Fixed `count()` not generating JOINs for navigation properties used in WHERE clauses:

```typescript
// Before: Crashed with "missing FROM-clause entry for table"
// After: Correctly joins the related table
const count = await db.orders
  .where(o => eq(o.user!.username, 'alice'))
  .count();
```

Also fixed multi-level navigation in `count()` (e.g., `ot.task!.level!.name`).

- Commits: [c340f71](https://github.com/brunolau/linkgress-orm/commit/c340f71), [d684e7a](https://github.com/brunolau/linkgress-orm/commit/d684e7a)
