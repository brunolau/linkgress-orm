## Feature: Query *from* a CTE — `db.selectFromCte()` with FULL OUTER / RIGHT / CROSS joins

### What

A query can now use a **CTE as its FROM root** (rather than an entity table) via
`db.selectFromCte(rootCte)`, unlocking join shapes the entity-anchored
`db.<table>.with(...).leftJoin(cte, …)` path cannot express — notably
`FULL OUTER` / `RIGHT` / `CROSS` joins and `ON TRUE` predicates between two CTEs.

```typescript
import { DbCteBuilder, eq, and, onTrue } from 'linkgress-orm';

const cteBuilder = new DbCteBuilder();

// A grouped/aggregate CTE body (one row per status) — now accepted by `with()`.
const spend = cteBuilder.with(
  'spend',
  db.orders
    .where(o => and(eq(o.userId, userId), eq(o.status, 'completed')))
    .select(o => ({ status: o.status, totalPrice: o.totalAmount }))
    .groupBy(o => ({ status: o.status }))
    .select(g => ({ status: g.key.status, totalPrice: g.sum(o => o.totalPrice) }))
);

const tier = cteBuilder.with(
  'current_tier',
  db.users
    .where(u => and(eq(u.id, userId), eq(u.isActive, true)))
    .select(u => ({ currentTierId: u.id }))
    .limit(1)
);

const rows = await db
  .selectFromCte(spend.cte)
  .fullOuterJoin(tier.cte, onTrue())
  .select((s, t) => ({
    status: s.status,
    totalPrice: s.totalPrice,
    currentTierId: t.currentTierId,
  }))
  .toList();
```

```sql
WITH "spend" AS (
  SELECT "orders"."status" as "status",
         CAST(SUM("orders"."total_amount") AS DOUBLE PRECISION) as "totalPrice"
  FROM "orders"
  WHERE ("orders"."user_id" = $1 AND "orders"."status" = $2)
  GROUP BY "orders"."status"
), "current_tier" AS (
  SELECT "users"."id" as "currentTierId"
  FROM "users"
  WHERE ("users"."id" = $3 AND "users"."is_active" = $4)
  LIMIT 1
)
SELECT "spend"."status" as "status",
       "spend"."totalPrice" as "totalPrice",
       "current_tier"."currentTierId" as "currentTierId"
FROM "spend"
FULL OUTER JOIN "current_tier" ON TRUE
-- params: [userId, 'completed', userId, true]
```

This preserves all four `FULL OUTER JOIN … ON TRUE` cases (both sides present,
spend-only with `NULL` tier, tier-only with `NULL` spend, or neither → zero rows).

### Added

- **`db.selectFromCte(rootCte)`** — starts a `CteRootQueryBuilder` whose FROM root is
  a `DbCte`.
- **`CteRootQueryBuilder` / `CteJoinedQueryBuilder`** — `innerJoin` / `leftJoin` /
  `rightJoin` / `fullOuterJoin` / `crossJoin`, plus `select`, `orderBy` (against output
  aliases), `limit`, `offset`, and `toList` / `first` / `toSql` / `buildQuery`. Joins
  attach a further CTE; `.select((root, …joined) => ({ … }))` projects the root plus
  each joined CTE (sources arrive in FROM declaration order).
- **`onTrue()`** — an always-true (`ON TRUE`) join predicate, for the cross-product
  form of `FULL OUTER JOIN` (which PostgreSQL requires an `ON`/`USING` clause for).
- **Grouped / aggregate CTE bodies in `DbCteBuilder.with()`** — `with()` now also
  accepts a `.groupBy(...).select(...)` body (`SUM` / `COUNT` / `MIN` / `MAX` +
  `GROUP BY`) as a *plain* CTE (one row per group), distinct from `withAggregation()`
  which folds the group into a `json_agg` array.
- New exports from `linkgress-orm`: `CteRootQueryBuilder`, `CteJoinedQueryBuilder`,
  `CteJoinType`, `onTrue`.

Every CTE body's parameters are emitted first, in `WITH` declaration order (root CTE,
then each joined CTE), followed by any `ON`-predicate parameters — so the whole
statement keeps a single, sequential `$1..$n` numbering.

### Compatibility

Fully backward compatible. All additions are new surface area: `selectFromCte()` is a
new entry point, the new builders/`onTrue()` are new exports, and `DbCteBuilder.with()`
merely *widens* the query bodies it accepts (existing `with()` / `withAggregation()` /
entity-rooted `.with(cte).leftJoin(…)` usage is unchanged).
