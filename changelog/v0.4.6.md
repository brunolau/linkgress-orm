## Fix: Missing JOINs for deep navigation in SELECT without intermediate field selection

When selecting only from the deepest level of a navigation chain (e.g., `p.product.resort.currency`) without also selecting fields from intermediate tables, the generated SQL was missing the necessary JOIN clauses, producing errors like `missing FROM-clause entry for table "resort"`.

**Before (broken):**
```typescript
db.cartItems
  .where(p => eq(p.cartId, cartId))
  .select(p => ({
    currency: p.product!.resort!.currency,
  }))
  .toList();
// Generated: SELECT "resort"."id" as "currency" FROM "cart_item" WHERE ...
// Error: missing FROM-clause entry for table "resort"
```

**After (fixed):**
```typescript
// Same query now correctly generates JOINs for all intermediate tables:
// SELECT "resort"."currency" as "currency"
// FROM "cart_item"
// LEFT JOIN "products" AS "product" ON ...
// LEFT JOIN "resorts" AS "resort" ON ...
// WHERE ...
```

### Root cause

`SelectQueryBuilder.collectTableAliasesFromSelection()` only collected `__tableAlias` (the leaf table) from FieldRefs but ignored `__navigationAliases` (intermediate navigation tables). The `GroupedSelectQueryBuilder` version of this method already had the fix, but the regular `SelectQueryBuilder` was missing it.

Existing tests passed by coincidence because they always selected fields from intermediate tables too, causing those aliases to be collected as leaf aliases.

### Files changed

- `src/query/query-builder.ts` — Added `__navigationAliases` collection in both the FieldRef and SqlFragment branches of `SelectQueryBuilder.collectTableAliasesFromSelection()`
- `tests/queries/multi-level-navigation.test.ts` — Added 3 tests for deep navigation without intermediate field selection
