## Fix: `toNumberList()` / `toStringList()` inside nested collections returns `[{}, ...]` when a sibling list is present

### What

When a parent `.select()` contains two or more sibling nested lists (e.g. `items.toList()` and `appliedCodes.toList()`), and one of those nested lists uses `.toNumberList()` or `.toStringList()` on a deep navigation walk, the flat list was returned as an array of empty objects (`[{}, {}, ...]`) instead of the expected primitive array.

This release fixes both the core transform bug and adds full multi-hop navigation support for nested collections in both the CTE and LATERAL strategies.

### Root cause

Two independent bugs combined to cause the failure:

**1. Transform bug** — `transformNestedCollectionValue()` iterated each element with `for (const key in item)`. Numbers have no enumerable own-keys, so every element collapsed to `{}`. The fix propagates `flattenResultType: 'number' | 'string'` through `nestedCollectionInfo` and short-circuits the object-mapping loop when the flag is set, returning the raw primitive array as-is.

**2. Multi-hop join bug (CTE)** — When a nested collection is accessed through two or more intermediate navigations (e.g. `cdc.discountCode!.discount!.discountProducts!`), the outer CTE joined the inner CTE with an incorrect column:

```sql
-- Wrong: discount_code_id ≠ discount_id (different tables)
LEFT JOIN cte_1 ON "cart_discount_codes"."discount_code_id" = cte_1.parent_id
```

The inner CTE's `parent_id` holds `discount_id` values (from `discount_products`), not `discount_code_id` values. The fix generates a scalar subquery that traverses the full navigation chain:

```sql
LEFT JOIN cte_1 ON (
  SELECT "discountCode"."discount_id"
  FROM "discount_codes" "discountCode"
  WHERE "discountCode"."id" = "cart_discount_codes"."discount_code_id"
) = cte_1.parent_id
```

**3. Multi-hop correlation bug (LATERAL)** — The same multi-hop chain in the LATERAL strategy used the first navigation step's FK column directly as the correlation value, which is wrong when there are two or more hops. The same scalar subquery pattern is applied to the correlated WHERE clause:

```sql
-- Wrong (1-hop assumption):
WHERE "discount_products"."discount_id" = "lateral_0_appliedCodes"."discount_code_id"

-- Correct (traverses the chain):
WHERE "discount_products"."discount_id" = (
  SELECT "discountCode"."discount_id"
  FROM "discount_codes" "discountCode"
  WHERE "discountCode"."id" = "lateral_0_appliedCodes"."discount_code_id"
)
```

### Acceptance test (from impl.md)

```ts
// Schema: Cart → CartDiscountCodes → DiscountCode → Discount → DiscountProducts
const raw = await db.carts
  .where(p => eq(p.uuid, cartUuid))
  .select(p => ({
    items: p.cartItems!.select(it => ({ id: it.id })).toList('items'),
    appliedCodes: p.cartDiscountCodes!.select(cdc => ({
      scopeProductIds: cdc.discountCode!.discount!.discountProducts!
        .select(dp => ({ id: dp.productId }))
        .toNumberList(),
    })).toList('appliedCodes'),
  }))
  .firstOrDefault();

// Now passes — previously returned [{}] per code
assert(raw.appliedCodes.every(c =>
  c.scopeProductIds.every(id => typeof id === 'number')
));
```

### Files Changed

- `src/query/query-builder.ts` — `processField()`: CTE join clause now uses a scalar subquery for `navPath.length >= 2` to correctly traverse multi-hop navigation chains; added `flattenResultType` propagation through `nestedCollectionInfo`; `transformNestedCollectionValue()` early-returns for primitive arrays
- `src/query/strategies/lateral-collection-strategy.ts` — `buildCorrelatedSubqueryAggregation()`: correlated WHERE clause uses scalar subquery for `collectionNavPath.length >= 2`
- `src/query/collection-strategy.interface.ts` — `nestedCollectionInfo.flattenResultType` field added; `collectionNavigationPath` field added to `CollectionAggregationConfig`
- `tests/queries/nested-to-number-list.test.ts` — 9 regression tests covering CTE strategy, LATERAL strategy, cross-strategy consistency, and the exact impl.md acceptance test (3-hop `DiscountCode → Discount → DiscountProducts` chain)
