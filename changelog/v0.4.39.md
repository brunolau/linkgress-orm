## Fix: Preserve custom-type mappers through `coalesce()` (and apply them in comparisons / `IN`)

Contributed by [@magicsk](https://github.com/magicsk) in [#13](https://github.com/brunolau/linkgress-orm/pull/13).

### What

A `coalesce()` over a column that has a custom type mapper used to drop that
mapper â€” the resulting `SqlFragment` carried no type, so:

- **reads** of the coalesced value skipped `fromDriver` (you got the raw driver
  value instead of your app type);
- **comparisons** against the coalesced expression (`eq`, `IN`, `NOT IN`, â€¦) did
  **not** run the operand through `toDriver`, so the wrong parameter value was
  sent;
- a **literal fallback** inside the coalesce (e.g. `coalesce(e.publishTime,
  someHourMinute)`) was passed through unconverted.

Now `coalesce()` propagates the first mapper found among its arguments onto the
resulting fragment, converts literal fallbacks with `toDriver`, and the fragment
carries the type both for result reading and for later comparisons.

```typescript
// publishTime uses a HourMinute <-> smallint custom mapper
db.posts
  .where(p => eq(coalesce(p.publishTime, { hour: 0, minute: 0 }), { hour: 9, minute: 30 }))
  .toList();
// The { hour: 9, minute: 30 } comparison value AND the { hour: 0, minute: 0 }
// fallback are now both run through the mapper's toDriver (â†’ smallint), and
// reading a coalesced publishTime applies fromDriver.
```

### How

`src/query/conditions.ts` gains three small helpers and rewires the value paths
to use them, so a "field with a mapper" now means either a `FieldRef`
(`__mapper`) **or** a `SqlFragment` (`getMapper()`), uniformly:

- `getValueMapper(value)` â€” resolves a mapper from a `FieldRef.__mapper` or a
  `SqlFragment.getMapper()`.
- `applyToDriverMapper(value, mapper)` â€” applies `mapper.toDriver(value)` when a
  mapper is present.
- `isSqlFragmentLiteral(value)` â€” distinguishes a plain literal fallback from
  another ref/fragment/placeholder, so only literals get converted.

Applied in:

- `coalesce()` â€” finds the first argument mapper, `toDriver`-converts literal
  fallbacks, and passes the mapper to the returned `SqlFragment` (so it preserves
  the type for both reads and comparisons).
- `WhereConditionBase` value formatting, and `InComparison` / `NotInComparison` â€”
  the compared literal(s) are now mapped via the field's mapper whether the field
  is a `FieldRef` or a mapper-bearing `SqlFragment` (previously only plain
  `FieldRef`s were handled).
